# ИНСТРУКЦИЯ ПО РАБОТЕ С ПРОЕКТОМ "AI chats export"

Этот документ содержит руководства:
1. По назначению и запуску всех скриптов автоматизации.
2. По самостоятельному повторению всего цикла импорта, проверки и восстановления данных.
3. По эффективному взаимодействию с ИИ-кодером (ограничение контекста и лимитов).

## 1. Файлы скриптов и их назначение

Все скрипты расположены в папке [src/](file:///C:/Codex_Personal/projects/AI%20chats%20export/src):

*   **`AI_chats_filter_optimized.py`**: Основной скрипт. Считывает сырые данные из `data/raw/`, бьет их на батчи по ~300 000 символов и отправляет в Gemini API. Производит первичную классификацию диалогов по категориям.
*   **`recover_lost_batches.py`**: Скрипт точечного восстановления. Используется, если при первом запуске гигантские батчи отвалились по таймауту 504. Он дробит эти батчи на мелкие под-блоки (до 100 000 символов) и отправляет их повторно. Если ИИ пришлет невалидный JSON, сохраняет сырой ответ в `Неразобранное_recovered.md`.
*   **`parse_unresolved_recovered.py`**: Разбирает `Неразобранное_recovered.md` с помощью Loose-парсера на регулярных выражениях и разносит данные по файлам.
*   **`parse_unresolved.py`**: Аналогичный парсер для разбора `Неразобранное.md` (куда складывались ответы с ошибками JSON при первичном запуске).
*   **`merge_small_files.py`**: Агрегатор категорий. Объединяет родственные мелкие файлы (например, `Финансы_и_Налоги`, `Финансы_и_Инвестиции` и т.д.) в крупные макро-категории (например, `Финансы_и_Право.md`).
*   **`clean_temporary_categories.py`**: Скрипт очистки временных файлов. Вырезает полезный текст из кривых категорий (например, `CategoryName.md`) и переносит в правильные файлы.
*   **`retry_failed_batches.py`**: Инструмент повторного прогона сбоящих батчей с ротацией моделей (использовался в Phase 2).

## 2. Как самостоятельно повторить весь цикл с нуля

Если вам нужно заново прогнать все архивы (например, при выгрузке свежего бэкапа чатов), выполните следующие шаги:

### Шаг 1. Подготовка
1. Переместите старые файлы из папки `data/done/` в архив, оставив папку пустой.
2. Положите новые сырые архивы (например, `conversations.json` и `МоиДействия.html`) в [data/raw/](file:///C:/Codex_Personal/projects/AI%20chats%20export/data/raw).

### Шаг 2. Первичный запуск
Запустите основной скрипт в терминале:
```bash
py src/AI_chats_filter_optimized.py
```
Скрипт разобьет файлы на батчи и начнет обработку. На выходе в папке `data/done/` создадутся MD-файлы категорий и лог `report_log.txt`.

### Шаг 3. Анализ лога на предмет жестких потерь
Откройте созданный файл `data/done/report_log.txt` и найдите в нем строки с ошибками `504` или `Deadline Exceeded`. Запомните номера упавших батчей (например, 15, 17, 44, 46).

### Шаг 4. Запуск восстановления утерянного
1. Откройте [src/recover_lost_batches.py](file:///C:/Codex_Personal/projects/AI%20chats%20export/src/recover_lost_batches.py).
2. Задайте список утерянных номеров батчей в переменной:
   `lost_batch_nums = [15, 17, 44, 46]`
3. Запустите скрипт восстановления:
   ```bash
   py src/recover_lost_batches.py
   ```

### Шаг 5. Постобработка неразобранного
1. Если после шага 2 образовался файл `Неразобранное.md`, запустите:
   ```bash
   py src/parse_unresolved.py
   ```
2. Если после шага 4 образовался файл `Неразобранное_recovered.md`, запустите:
   ```bash
   py src/parse_unresolved_recovered.py
   ```

### Шаг 6. Очистка и Финальное слияние
1. Выполните перенос из временных файлов:
   ```bash
   py src/clean_temporary_categories.py
   ```
2. Объедините все мелкие категории в крупные макро-файлы:
   ```bash
   py src/merge_small_files.py
   ```

Все файлы базы знаний в `data/done/` готовы к загрузке в NotebookLM!

## 3. Инструкция по эффективной работе с ИИ-кодером (Antigravity)

При постановке задач ИИ-ассистенту (Gemini/Cursor/Antigravity) в чате обязательно придерживайтесь следующих правил, чтобы не расходовать лимиты контекста и предотвратить генерацию лишнего кода:

1.  **Запрет на рекурсивное чтение файлов:** 
    *Промпт:* *«Не читай файлы кода целиком. Если тебе нужно изучить код, читай только конкретные строки или импортируемые функции через инструмент `view_file` с указанием диапазона строк (StartLine/EndLine).»*
2.  **Запрет на создание кода с нуля без крайней необходимости:**
    *Промпт:* *«Не переписывай скрипты заново. Вноси точечные изменения в существующие файлы с помощью инструмента `replace_file_content`.»*
3.  **Использование готовых индексов и паспорта проекта:**
    *Промпт:* *«Перед началом работы изучи [PROJECT_HANDOFF.md](file:///C:/Codex_Personal/projects/AI%20chats%20export/PROJECT_HANDOFF.md) и [progress.md](file:///C:/Codex_Personal/projects/AI%20chats%20export/progress.md). Используй их как единый источник правды о структуре и статусе проекта. Не сканируй заново логи и директории без явного указания.»*
4.  **Кодировка Cyrillic:**
    *Промпт:* *«Все создаваемые или редактируемые Markdown/текстовые файлы, содержащие кириллицу, сохраняй строго в кодировке UTF-8 с BOM (utf-8-sig).»*
