## 2. Технические нюансы и извлеченные уроки

*   **Кодировки в логах:** Исходный файл `report_log.txt` имел смешанную кодировку (ANSI/CP866 для системных выводов PowerShell и UTF-8 для логов Python). Отладка парсинга логов производилась с параметром `errors='replace'` в `open()` для предотвращения падений.
*   **Кодировка результатов:** Все MD-файлы в папке `data/done/` строго сохранены в кодировке **UTF-8 с BOM (`utf-8-sig`)**, чтобы IDE и текстовые редакторы на Windows корректно отображали кириллицу.
*   **Баг "Ошибочного статуса" в аудите логов:** Старый скрипт аудита логов читал сообщения о падении следующего батча из-за Lookahead-окна в 25 строк и ошибочно ставил статус "Failed" предыдущему батчу. Баг исправлен добавлением жесткого `break` при обнаружении заголовка следующего батча в анализируемом окне.
*   **Буферизация Stdout:** При фоновом запуске задач в Windows Python буферизует `stdout`. Из-за этого лог-файл задачи (`task-*.log`) остается пустым до тех пор, пока процесс не завершится. Мониторинг активности выполняющегося скрипта производился с помощью утилиты `netstat -ano | findstr <PID>`, которая показывала активное сетевое соединение `ESTABLISHED` с серверами Google API (порт 443).
