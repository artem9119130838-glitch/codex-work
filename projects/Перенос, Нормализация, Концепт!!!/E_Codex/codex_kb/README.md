# Codex Knowledge Base

Это долговременная база знаний по повторяющимся направлениям работы. Она не заменяет проекты, а дает общие постоянные факты, правила безопасности, архитектурные решения и проверенные процедуры.

## Структура

```text
codex_kb
  00_global        общие правила, карта направлений, стиль ведения знаний
  10_assets        постоянные объекты: сервер, компьютер, телефон, роутер
  20_domains       направления работ: 1С, n8n, Bitrix, тендеры, почта
  30_runbooks      повторяемые операционные процедуры
  40_prompts       промпты для новых чатов и проектных стартов
  50_skills_index  индекс повторяемых навыков из старых чатов
  progress         прогресс по доменам
  skills           локальные чеклисты и навыки
  90_sources       карта источников и старых экспортов
  templates        шаблоны файлов для новых проектов
```

## Правило источников

- Стабильный факт живет в `00_global`, `10_assets` или `20_domains`.
- Операционная процедура живет в `30_runbooks`.
- Проектная конкретика живет в папке проекта.
- Сырые старые чаты остаются в `E:\Codex_Work\chat_exports`.
- Если факт сомнителен, писать `unknown` и ссылаться на источник.

## Как обновлять

После крупного результата:

1. Обновить доменный файл в `20_domains`.
2. Если добавился новый постоянный объект, обновить `10_assets`.
3. Если появилась повторяемая процедура, добавить или обновить runbook.
4. Обновить `E:\Codex_Work\PROGRESS.md` и доменный progress-файл.
5. Если появился новый тип чата, добавить промпт в `40_prompts`.
6. Если изменились `00_global`, `20_domains` или важные runbook-и, проверить `RAG_SYNC_RUNBOOK.md`.
7. Не копировать секреты.
