# Security Rules

## Секреты

Не записывать в markdown:

- пароли 1С;
- пароли серверов;
- Bitrix webhook URL целиком, если в нем есть секрет;
- Google OAuth client secret;
- n8n encryption key;
- WireGuard private keys;
- SSH private keys;
- cookies/session tokens.

Можно записывать:

- домен без секрета;
- имя сервиса;
- путь к месту, где секрет хранится;
- имя пользователя без пароля;
- последние 4 символа токена для сверки, если очень нужно.

## Сервер

VPS считается production-like:

- не перезапускать 1С и PostgreSQL без подтверждения;
- перед Apache изменениями делать `apachectl configtest`;
- предпочитать `systemctl reload apache2`;
- не делать массовые cleanup-команды без явной причины;
- не закрывать 1С порты до проверки VPN-доступа.

## Запрещенные удаления без отдельного подтверждения

- `/var/1C/licenses`
- `/Storage/docker-data`
- `/Storage/home`
- `/Storage/data`
- рабочие папки с бэкапами
- старые chat exports

## 1С

- Не править SQL-таблицы 1С напрямую без бэкапа и понимания метаданных.
- Не запускать тяжелое тестирование и исправление 1С без окна обслуживания.
- OData давать через отдельного пользователя с минимальными правами.

## n8n

- Не хранить credentials в workflow заметках.
- Использовать credentials store n8n.
- Для опасных интеграций сначала делать dry-run.
- Для массовых операций добавлять логирование и защиту от дублей.
