## Оценка предложенного плана (кратко)

План в целом верно определяет болевые точки (полное чтение файлов, бесконечные циклы, отсутствие учета стоимости) и предлагает разумные меры. Однако он **недостаточно агрессивен** для достижения *максимальной* экономии токенов и предварительного расчета. Отсутствуют жесткие лимиты, автоматизация оценки, принудительное ограничение контекста и защита от человеческого фактора.

## Необходимые изменения и дополнения

### 1. Калькулятор токенов → автоматический пре-челленджер

**Проблема**: ручной запуск `token_cost_estimator.py` будут игнорировать.  
**Решение**: создать **обязательный pre-flight скрипт**, который:

- Перехватывает промпт пользователя перед отправкой в API (можно через прокси или обёртку CLI).
- Подсчитывает токены локально (tiktoken или аналог для Gemini).
- Если оценка превышает порог (например, $0.01 за запрос), требует подтверждения.

```python
# Пример порогов в конфиге
COST_LIMIT_PER_REQUEST = 0.005  # $0.005
TOKEN_LIMIT_INPUT = 32000        # жесткое ограничение
```

**Добавить в план**:
- Разработка `preflight_hook.py`, интегрируемого в оболочки вызовов ИИ (CLI, плагин IDE).
- Для Antigravity – скрипт-обёртка, заменяющая прямые вызовы API.

### 2. Глобальный регламент экономии – ужесточить количественно

В `AGENTS.md` добавить **цифровые лимиты**, которые ассистент не может превысить (должны быть записаны в системный промпт):

| Действие | Лимит | Примечание |
|----------|-------|-------------|
| Чтение файла через `view_file` | максимум 200 строк за раз | При необходимости больше – использовать несколько вызовов с паузой |
| Глубина `grep_search` рекурсивно | 3 уровня вложенности | Для больших проектов явно указывать `--max-depth` |
| Количество файлов в одном `grep` | не более 20 | Иначе раздувается вывод |
| Итерации fail-fast | 3 | После 3 – выдать короткую ошибку и остановиться |
| Токенов в ответе ассистента | 4000 (включая код) | Принудительное усечение через `max_output_tokens` API |
| Длина истории чата, передаваемой субагенту | последние 2000 токенов | Без полной истории |

**Дополнительно**: Запретить ассистенту вызывать `read_file` без указания `start_line/end_line` для любых файлов >50 строк.  
**Механизм контроля**: В системный промпт добавлять строку: *"Ты обязан соблюдать лимиты из раздела TOKEN_LIMITS. Нарушение приведет к принудительной остановке."*

### 3. Предварительный расчет использования токенов – встроить в CI/CD

План упоминает оценку стоимости, но не интегрирует её в процесс разработки. **Добавить**:

- **Pre-commit хук**: при изменении любого `.md` или `.py`, содержащего промпты для ИИ, запускается `token_cost_estimator.py` и выводит предупреждение, если ожидаемый расход на типовую сессию > $0.10.
- **GitHub Actions (или аналог)**: еженедельный отчёт по фактическому потреблению токенов из логов API (если логируются).
- **Локальный дашборд**: простой HTML, парсящий логи `.jsonl` и показывающий топ-10 самых дорогих запросов.

### 4. Инструкция для вайб-кодеров – конкретные шаблоны промптов

В `README_HOW_TO_USE.md` заменить общие рекомендации на **жёсткие шаблоны** и примеры:

**Плохой промпт** (расходует токены):  
*«Посмотри на весь проект и исправь баги в логине»*

**Хороший промпт** (экономит):  
*«В файле auth/login.py (строки 45-78) функция validate_token() возвращает False. Покажи diff с исправлением. Не читай другие файлы.»*

Добавить **чек-лист перед отправкой запроса**:
- [ ] Указаны конкретные файлы и диапазоны строк  
- [ ] Запрос не требует сканирования `node_modules` или `dist`  
- [ ] Явно запрещён вывод полных файлов в ответ  
- [ ] Модель выбрана **Flash**, если не требуется сложная архитектурная логика  

### 5. Контроль глубины поиска – автоматическая фильтрация

Добавить в проект файл `.aiignore` (по аналогии с `.gitignore`), который ассистент **обязан** проверять перед любым рекурсивным обходом. Содержимое по умолчанию:

```
node_modules/
dist/
build/
*.log
*.tmp
.git/
.cache/
__pycache__/
*.pyc
```

В `AGENTS.md` прописать: *«Перед вызовом grep_search или read_dir – проверить наличие .aiignore и исключить все перечисленные пути. Нарушение считается критической ошибкой.»*

### 6. Превентивные меры для платных API – добавить бэкап-модель и batch-режим

**Flash-First** – это хорошо, но нужно добавить:

- **Fallback-стратегию**: если Flash возвращает `429` (rate limit) или слишком много ошибок – переходить на **Gemini 1.5 Flash-8B** (ещё дешевле) или локальную модель (например, Llama 3 8B через Ollama) для некритичных задач.
- **Batch API** для задач, не требующих реального времени: генерация документации, рефакторинг по чёткому паттерну, добавление логов. У Gemini batch-режим даёт скидку 50% (если есть – уточнить, в план добавить как рекомендацию).
- **Context Caching** – обязателен для проектов с базой знаний >20k токенов. Добавить в план конкретный скрипт `setup_cache.py`, который предварительно загружает и замораживает контекст.

### 7. Мониторинг расходов в реальном времени

Создать `token_budget_tracker.py`, который:

- Запускается вместе с сессией ИИ-ассистента.
- Перехватывает каждый вызов API (через подмену `requests.post` или переменную `HTTP_PROXY`).
- Суммирует стоимость.
- При достижении лимита (например, $0.50 за сессию) **принудительно завершает процесс** и пишет `STOP: бюджет исчерпан`.

В `AGENTS.md` добавить: *«Ассистент не имеет права игнорировать бюджетный трекер. Если трекер остановил сессию – завершить работу.»*

### 8. План верификации – добавить стресс-тесты на перерасход

Существующие тесты (работа калькулятора, кодировка) – недостаточны. **Добавить**:

- **Тест на принудительное соблюдение лимитов**: искусственно послать запрос, требующий прочитать 100 файлов, проверить, что ассистент отказался или прочитал не более 20.
- **Тест на fail-fast**: дать заведомо неисправимую синтаксическую ошибку, убедиться, что после 3 итераций ассистент остановился и не сжёг токены на четвёртую.
- **Тест на предварительный расчёт**: отправить промпт с оценкой 0.02$, проверить, что pre-flight скрипт запросил подтверждение.

| 3 | .aiignore и обязательная его проверка | 🔴 High |
| 4 | Бюджетный трекер в реальном времени | 🟠 Medium |
