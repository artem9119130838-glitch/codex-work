2. Создание глобального регламента экономии для ИИ-ассистентов
[NEW] 
AGENTS.md
Создадим глобальный файл инструкций для ИИ-ассистентов (включая Antigravity) в общем контуре. В нем жестко зафиксируем следующие правила экономии:

Экономное чтение файлов: Использовать view_file с аргументами StartLine и EndLine для любых файлов длиннее 100 строк. Запретить чтение файлов целиком, если требуется локальное изменение или анализ конкретной функции.
Целевой поиск: Использовать grep_search с фильтрами по расширениям вместо последовательного просмотра файлов в папках.
Fail-fast при ошибках: Ограничить автоматические итерации по исправлению синтаксических ошибок или тестов до 3-х попыток. Если ошибка не исправлена, ассистент должен остановиться и обратиться к инженеру, приложив только краткий лог ошибки, а не весь вывод сборщика.
Очистка контекста субагентов: При запуске субагентов через invoke_subagent передавать им строго минимальный контекст (без истории основного чата), необходимый для решения их локальной задачи.
Запрет на дублирование: Не выводить длинные листинги кода в чат, если они уже записаны в файлы. Заменять их краткими диффами или ссылками.

План предотвращения перерасхода API токенов при переходе на платные модели (Превентивные меры)
Мониторинг расходов:
Внедрение привычки оценки стоимости сессии перед сложными операциями с помощью созданного скрипта token_cost_estimator.py.
Context Caching:
Для больших проектов с базой знаний (>32k токенов) при вызовах через API использовать механизм Context Caching. Это сокращает стоимость входных токенов до 4 раз, если один и тот же контекст (база знаний, структура проекта) передается во многие запросы подряд.
Flash-First подход:
По умолчанию использовать Gemini 2.0 Flash / Gemini 1.5 Flash. Переключаться на Gemini 1.5 Pro только вручную при явной необходимости (например, сложные рефакторинги, затрагивающие множество файлов одновременно).
Контроль глубины поиска:
Ограничить глубину обхода директорий при поиске и анализе проекта. Запретить ассистенту сканировать папки рекурсивно без четко заданных масок файлов.
