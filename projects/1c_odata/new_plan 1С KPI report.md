# План Proof of Concept (PoC) по переиспользованию типовой калькуляции (1С:УНФ 3.0)

Мы согласны с вашими опасениями. Прежде чем перестраивать весь отчет и закладывать архитектурные решения, необходимо на практике проверить:
1. Выполняется ли расчет и заполнение таблицы `КалькуляцияНаФорме` при вызове `ПолучитьФорму()` на сервере без ее интерактивного открытия.
2. Достаточно ли переданного набора параметров для корректной инициализации формы.
3. Содержится ли в процедурах общего модуля аналогичный метод, позволяющий избежать инициализации формы (исследование кода показало, что расчетные запросы зашиты непосредственно в форму, но мы перепроверим общие модули).

---

## 1. План проведения PoC

### Шаг 1. Создание тестовой внешней обработки
Пользователь создает в Конфигураторе 1С простую внешнюю обработку со следующими параметрами:
- **Реквизиты**:
  - `Заказ` (тип `ДокументСсылка.ЗаказПокупателя`)
- **Форма**:
  - Создать основную форму обработки, вынести на нее реквизит `Заказ` и добавить команду (кнопку) `ЗапускТеста`.

### Шаг 2. Код модуля формы обработки
Вставить следующий BSL-код в модуль формы тестовой обработки:

```bsl
&НаКлиенте
Процедура ЗапускТеста(Команда)
	ТестНаСервере();
КонецПроцедуры

&НаСервере
Процедура ТестНаСервере()
	
	Если Не ЗначениеЗаполнено(Заказ) Тогда
		Сообщить("Выберите заказ покупателя для теста!");
		Возврат;
	КонецЕсли;
	
	ЗаказОбъект = Заказ.ПолучитьОбъект();
	
	// 1. Формируем структуру данных для формы калькуляции
	СтруктураДанных = Новый Структура;
	СтруктураДанных.Вставить("ТолькоПросмотр", Истина);
	СтруктураДанных.Вставить("Ссылка", Заказ);
	СтруктураДанных.Вставить("ТекущийВариантКП", ?(ЗаказОбъект.КоличествоВариантовКП > 0, ЗаказОбъект.ОсновнойВариантКП, 0));
	СтруктураДанных.Вставить("СкидкаПроцент", 0);
	СтруктураДанных.Вставить("СкидкаСумма", 0);
	
	ДанныеЗаказа = Новый Структура;
	// Заполняем реквизиты шапки документа
	ИменаРеквизитов = "Организация,Дата,ВидОперации,СпособРасчетаСебестоимостиКалькуляции,Договор,Курс,Кратность,ВалютаДокумента,ОсновнойВариантКП,КоличествоВариантовКП,КалькуляцияРассчитана,ШаблонКалькуляции,КомментарийКалькуляции,РасчетКалькуляцииПоДаннымТЧ";
	ЗаполнитьЗначенияСвойств(ДанныеЗаказа, ЗаказОбъект, ИменаРеквизитов);
	СтруктураДанных.Вставить("ДанныеЗаказа", ДанныеЗаказа);
	
	// Выгружаем табличные части
	СтруктураДанных.Вставить("Запасы", ЗаказОбъект.Запасы.Выгрузить());
	СтруктураДанных.Вставить("Работы", ЗаказОбъект.Работы.Выгрузить());
	СтруктураДанных.Вставить("Материалы", ЗаказОбъект.Материалы.Выгрузить());
	СтруктураДанных.Вставить("Исполнители", ЗаказОбъект.Исполнители.Выгрузить());
	СтруктураДанных.Вставить("Калькуляция", ЗаказОбъект.Калькуляция.Выгрузить());
	СтруктураДанных.Вставить("ПараметрыДоставки", ЗаказОбъект.ПараметрыДоставки.Выгрузить());
	СтруктураДанных.Вставить("ВидыЦен", Новый Массив);
	
	АдресДанных = ПоместитьВоВременноеХранилище(СтруктураДанных);
	ПараметрыФормы = Новый Структура("АдресДанных", AddressДанных);
	
	// Имя формы в УНФ: "Документ.ЗаказПокупателя.Форма.ФормаКалькуляция"
	// (Пожалуйста, проверьте в Конфигураторе в метаданных документа ЗаказПокупателя точное имя формы калькуляции)
	ИмяФормыКалькуляции = "Документ.ЗаказПокупателя.Форма.ФормаКалькуляция"; 
	
	// 2. Вызываем ПолучитьФорму() на сервере
	Попытка
		ФормаКальк = ПолучитьФорму(ИмяФормыКалькуляции, ПараметрыФормы);
		
		Сообщить("Успешно создали форму калькуляции на сервере!");
		
		// 3. Выгружаем КалькуляцияНаФорме
		ТаблицаКалькуляции = ФормаКальк.КалькуляцияНаФорме.Выгрузить();
		
		Сообщить("Количество строк в КалькуляцияНаФорме: " + ТаблицаКалькуляции.Количество());
		
		// Выводим тестовые строки для проверки сопоставления плана и факта
		Для Каждого Строка Из ТаблицаКалькуляции Цикл
			Сообщить(СтрШаблон("Статья/Номенклатура: %1 | План себест: %2 | Факт себест: %3", 
				Строка.Номенклатура, Строка.Себестоимость, Строка.СебестоимостьФакт));
		КонецЦикла;
		
	Исключение
		Сообщить("Ошибка вызова формы: " + ОписаниеОшибки(), СтатусСообщения.Важное);
	КонецПопытки;
	
КонецПроцедуры
```

### Шаг 3. Запуск теста
Пользователь запускает обработку в режиме "1С:Предприятие", выбирает заказ (желательно с закрытым месяцем, по которому есть фактические расходы) и нажимает кнопку.

## 2. Анализ результатов PoC

Мы ожидаем два варианта исхода теста:

### Сценарий А (Успех)
Таблица `КалькуляцияНаФорме` заполнена, строки выведены с плановыми и фактическими показателями. 
*Действие*: Мы переходим к реализации внешнего отчета по этой архитектуре.

### Сценарий Б (Таблица пустая / Ошибка инициализации)
Таблица пустая или возникла критическая ошибка из-за отсутствия контекста интерфейса.
*Действие*: Мы перейдем к анализу общих модулей и выделим точные запросы сбора факта и расчета в модуль отчета (как и предлагалось во втором варианте).
