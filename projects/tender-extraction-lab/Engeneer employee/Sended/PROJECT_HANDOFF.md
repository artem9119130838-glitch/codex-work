## 1. Смысл
Экспериментальный проект (песочница) для автоматического извлечения ключевых данных из папок тендеров (PDF/DOCX/XLSX) и интеграции с Битрикс24.

## 2. Текущее техническое состояние
- Разработчик Михаил Денисов спроектировал и собрал прототип микросервиса **Tender RAG API** на FastAPI (архив `tender-rag-api.zip` в каталоге `Engeneer employee`), который включает:
  * Парсинг документов через `MarkItDown`.
  * Семантическую нарезку на чанки через `RecursiveCharacterTextSplitter`.
  * Локальную векторизацию с помощью модели `sentence-transformers` (`intfloat/multilingual-e5-small`).
  * In-Memory векторную базу данных для косинусного поиска кандидатов.
  * Регулярные выражения для прямого извлечения ИНН и сумм (НМЦК).
- Локальное тестирование микросервиса на макбуке разработчика с проксированием через `Ngrok` прошло успешно. Проверен сквозной триггер Google Drive -> n8n -> FastAPI API.

## 3. Блокер архитектуры / Развертывания на VPS
- **Проблема с ОЗУ:** Локальный запуск модели векторизации `sentence-transformers` требует загрузки PyTorch и весов модели, что потребляет ~1.5 ГБ RAM. На сервере VPS (где крутятся 1С, Postgres Pro, VPN и n8n) свободно лишь около 1 ГБ RAM при 5 активных пользователях 1С. Попытка развернуть этот контейнер на VPS приведет к OOM-аварии сервера.
- **Ограничение n8n нод:** Выполнить весь код внутри встроенной Python-ноды n8n невозможно из-за ограничений песочницы Pyodide (невозможность компиляции C-зависимостей вроде `torch` и хранения тяжелых весов модели в памяти).
