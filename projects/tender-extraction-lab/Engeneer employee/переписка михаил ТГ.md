LLM получает:
Поле: Оплата
Кандидат 1...5 ↓
Возвращает:
{ "payment_terms": "...", "confidence": "high", "source_file": "...", "source_url": "...", "source_page": 17 }

То же самое отдельно для: → поставки → обеспечения → номенклатуры → ограничений → требований к участнику

При этом ИНН, заказчик, сумма, номер закупки, контакты и другие формальные поля желательно доставать обычным кодом без LLM, чтобы не тратить токены.

То есть логика должна быть такая:
Google Drive ↓ Markdown ↓ Chunks ↓ Keyword Search + Embeddings ↓ Кандидаты по каждому полю ↓ LLM только для сложных полей ↓ JSON ↓ Google Sheet / Bitrix24

Основная мысль: не читать моделью весь тендер и не пытаться сразу строить идеальный RAG.
