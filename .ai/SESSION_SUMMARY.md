## 3. Анализ ошибок и уроки (Lessons Learned)
* **Проблема прав pg_dump:** При создании резервных копий от имени системного пользователя `postgres`, утилита `pg_dump` падала с ошибкой `Permission denied`, если папка бэкапа была создана пользователем `root`.
  * *Решение:* Всегда явно передавать права на папки дампов пользователю СУБД: `chown -R postgres:postgres <директория_бэкапа>`.
* **Блокировка push из-за больших файлов:** GitHub отклонял отправку коммитов (`pre-receive hook declined`) из-за тяжелых файлов в локальной истории (`1Cv8.cf` на 500 MB и `tender-rag-api.zip` на 406 MB).
  * *Решение:* Сделать мягкий сброс `git reset --soft origin/master`, исключить тяжелые файлы из отслеживания Git `git rm --cached <путь>` (оставив их физически на диске), прописать их в локальный список исключений `.git/info/exclude` и сделать чистый коммит.

## 4. Памятка для других чатов и следующие шаги
Поскольку пользователю потребовалось выключить интернет, на сервере был запущен автономный фоновый процесс бэкапа (PID `3111082`). 
Когда интернет снова появится, следующему ИИ-ассистенту необходимо:

1. **Скачать готовый бэкап с сервера на локальный диск D:**
   ```powershell
   scp -r -i C:\Users\Артем\.ssh\id_ed25519_wlisses root@109.248.170.181:/Storage/full-backup-2026-07-12_* D:\server-backups\
   ```
2. **Очистить временную папку бэкапа на сервере VPS:**
   ```powershell
   ssh -i C:\Users\Артем\.ssh\id_ed25519_wlisses root@109.248.170.181 "rm -rf /Storage/full-backup-2026-07-12_*"
   ```
