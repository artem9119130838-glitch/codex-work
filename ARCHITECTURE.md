# AI Rules & Security Architecture (Personal)

Этот файл описывает архитектуру ИИ-контракта, логику взаимодействия файлов правил и приоритеты их применения.

## Схема взаимодействия файлов

```
[MANIFEST.md] (Краткие метаданные проекта)
       ↓
[AGENTS.md] (Точка входа / ИИ-контракт)
  ┌────┴──────────────────────────┐
  ▼                               ▼
[AI_RULES.md] (TOKEN-FIRST)     [SECURITY.md] (Decision Policy L0-L6)
  └────┬──────────────────────────┘
       ▼
[SKILLS.md] (Индекс навыков)
       ▼
[Глобальные навыки] (windows, linux, git, 1c_unf, n8n, excel, session_management)
```

## Приоритет правил при конфликтах

Если правила в разных файлах противоречат друг другу, применяется следующий приоритет:

1. **[SECURITY.md](file:///C:/Codex_Personal/SECURITY.md)** — Матрица рисков и ограничений безопасности (высший приоритет).
2. **[AGENTS.md](file:///C:/Codex_Personal/AGENTS.md)** — Базовый ИИ-контракт и идентичность.
3. **[AI_RULES.md](file:///C:/Codex_Personal/AI_RULES.md)** — Регламенты TOKEN-FIRST и ведения логов.
4. **Правила конкретных проектов** (внутри папок проектов).
5. **Глобальные навыки** (Skills).

## Порядок применения правил ИИ-агентом

1. При запуске сессии ИИ читает **[MANIFEST.md](file:///C:/Codex_Personal/MANIFEST.md)** и **[AGENTS.md](file:///C:/Codex_Personal/AGENTS.md)** для определения базового контекста.
2. В процессе работы, если выполняются потенциально опасные команды (изменения, удаления, пуши), ИИ обращается к **[SECURITY.md](file:///C:/Codex_Personal/SECURITY.md)** для проверки уровня риска и необходимости запроса подтверждения.
3. Для оптимизации токенов перед открытием файлов ИИ использует правила **[AI_RULES.md](file:///C:/Codex_Personal/AI_RULES.md)**.
4. При возникновении типовых задач (работа с Acrobat, Linux, Git, 1С), ИИ находит нужный навык в **[SKILLS.md](file:///C:/Codex_Personal/SKILLS.md)** и считывает только его файл `SKILL.md` из глобального каталога навыков.
